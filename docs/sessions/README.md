# docs/sessions/ — Session logs

Session logs are the technical record of what happened. Created at session start, filled during the work, closed at session end. Durable findings are extracted to their permanent home (spec, research, operational memory) with a pointer left here; the log is the narrative, not the archive.

**Naming:** `YYYY-MM-DD-NNN-topic-slug.md`
**Status lifecycle:** In Progress → Completed

See `docs/README.md` for the canonical session log template, section matrix, and line-1 index conventions.
