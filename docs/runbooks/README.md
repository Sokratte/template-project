# Runbooks

Runbooks are step-by-step operational guides. They answer: "How do I do X?"
or "What do I do when Y happens?"

Examples of runbooks:
- How to deploy the service to production
- What to do when the database runs out of space
- How to rotate API keys or secrets
- How to recover from a failed deployment
- How to onboard a new contributor

Each runbook is a single markdown file. Name it descriptively:
`deploy-to-production.md`, `rotate-api-keys.md`, `database-recovery.md`.

This directory is empty in the template. Add runbooks as your project
develops operational procedures.
