# docs/decisions/ — Architecture Decision Records

Decision records (ADRs) capture what was decided and why. They are durable, append-only: once accepted, the reasoning stays. Substantive changes append a dated `## Addendum` and mark superseded passages `~~SUPERSEDED~~`; the old reasoning is never deleted.

**Naming:** `ADR-NNN-topic-slug.md`
**Status lifecycle:** Draft → Accepted → Superseded → / Deprecated

See `docs/README.md` for the canonical ADR template, section matrix, and line-1 index conventions.
