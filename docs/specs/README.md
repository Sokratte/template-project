# docs/specs/ — Specifications

Specs define what "done" looks like. Code that contradicts an active spec is a bug. They are the living reference for what is being built; they are not archived when implemented.

**Naming:** `SPEC-NNN-topic-slug.md`
**Status lifecycle:** Draft → Active → Superseded →

See `docs/README.md` for the canonical spec template, section matrix, and line-1 index conventions.
