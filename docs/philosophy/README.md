# docs/philosophy/ — Design philosophies

Design philosophies (DPs) capture foundational principles — not decisions about what to build, but stances on how to think. Ratified DPs (Status: Accepted) are immutable: to change one, supersede it and move the old file to `docs/philosophy/archive/` under its dated name.

A DP never references specs, ADRs, or plans. It never points down; other documents point up to it.

**Naming:** `DP-NNN-topic-slug.md`
**Status lifecycle:** Draft → Accepted → Superseded →  / Deprecated

See `docs/README.md` for the canonical DP template, section matrix, and line-1 index conventions.
